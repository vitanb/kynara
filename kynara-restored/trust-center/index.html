Security architecture, in one paragraph

Kynara is built fail-closed: when the policy engine cannot reach the database, decisions return deny. Tenants are isolated with PostgreSQL Row-Level Security (per-request app.org_id session GUC) and per-tenant data keys via KMS — Enterprise plans support BYOK with customer-managed keys. Audit logs are SHA-256 hash-chained with append-only triggers and a customer-runnable offline verifier. AuthN is Argon2id passwords + MFA, or Okta/Azure-AD via SAML 2.0 / OIDC with PKCE. Decisions are logged with the matched policy and a human-readable reason to satisfy GDPR Article 22.
